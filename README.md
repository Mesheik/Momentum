# Momentum
## Elevator Pitch

Momentum is a full-stack web application that helps users overcome decision avoidance by turning overwhelming tasks into small, manageable actions. It tracks how long decisions are delayed and guides users to take their first step, helping them build momentum and move forward.

## Problem Statement

Many people don’t struggle because they can’t make decisions—they struggle because they avoid them. Important tasks like applying for jobs, studying, or handling responsibilities often get delayed due to overwhelm, uncertainty, or lack of direction.

Over time, these delays create stress, missed opportunities, and a lack of progress. While most productivity tools focus on completing tasks, they fail to address the real problem: helping users start.

Momentum exists to solve that starting point.

## Target User

Momentum is designed for students, early-career professionals, and individuals who frequently delay important decisions due to feeling overwhelmed. This includes users preparing for jobs, learning new skills, or managing responsibilities who need a structured way to move from avoidance to action.

## Why This Project

I chose to build Momentum because I have personally experienced the challenge of delaying important decisions like job applications, skill development, and personal goals. I understand how difficult it can be to take the first step, even when you know what needs to be done.

This project focuses on solving that exact problem by helping users move from inaction to progress in a realistic and sustainable way.

## Core Features
### 1. Decision Tracking
Users can log decisions they are avoiding
Each decision includes:
title
description
date created
### 2. Delay Tracking System
Tracks how long a decision has been postponed
Displays delay in days
Highlights decisions that have been avoided the longest
Creates awareness around avoidance patterns
### 3. Action Breakdown System
Each decision is broken into small, manageable steps
Designed to reduce overwhelm and make starting easier
Helps users move from thinking → doing

Examples:

“Open resume”
“Apply to one job”
“Review one topic”
### 4. MicroWins (Momentum Builder)
Small completed steps are treated as MicroWins
Users can mark steps as complete
Each MicroWin contributes to building momentum

Benefits:

reduces pressure
builds confidence
encourages continued action
### 5. Progress Tracking
Tracks completed MicroWins
Shows progress toward completing each decision
Reinforces consistency and forward movement

## What Makes Momentum Different

Most productivity tools focus on what users should do.
Momentum focuses on what users are avoiding.

Instead of overwhelming users with large tasks, Momentum helps them take small, achievable steps that lead to real progress.

## Project Goal

The goal of Momentum is to help users overcome avoidance, understand the impact of delaying decisions, and take meaningful action by building momentum through small, achievable steps.
